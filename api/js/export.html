# ES6模块导入导出

# 导入

  • 只想导入源模块的部分内容 假设a.js为源模块
// a.js
export let num = 10
export let name = '张三'

需要在b.js中使用a模块中的num

// b.js
import { num } from './a'

console.log(num) 		// 10

  • 重命名 假设a.js为源模块
// a.js
export let tademingzijiaozhangsan = '张三'

看到这里，我都忍不住要骂人，奈何咱没权限修改模块，只能重命名了

// b.js
import { tademingzijiaozhangsan as name } from './a'

console.log(name) 		// 张三

  • 想把模块中的所有导出都导入进来，但模块偏偏不是以export default方式导出 假设a.js为源模块
// a.js
export let num = 10
export let name = '张三'

在b.js中，只需要使用通配符*就可以把a.js中的模块都导入进来

// b.js
import * as moduleA from './a'

console.log(moduleA.num) 		// 10

  • 源模块没有对外任何导出
// a.js
let arr = [1,2,3]
arr.push(4)
console.log(arr)

在b.js中引入整个a.js就可以了

// b.js
import './a' 		// 会执行a.js中的所有逻辑

# 导出

  • 导出部分内容
// 导出
export let num = 10

// 导入
import { num } from './a'
  • export default
// 导出
const getName = () => {
	let name = '张三'
	return name
}
export default {
	getName
}

// 导入
import a from './a'
a.getName() 		// 张三
